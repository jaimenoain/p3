## Route groups (Public / Authenticated / Guest)

Route groups isolate session-dependent experiences; layouts are pass-through and establish boundaries only.

### App router file tree

```
app/
├── layout.tsx              # Root layout (fonts, globals)
├── page.tsx                # Landing /
├── globals.css
├── favicon.ico
├── (auth)/                 # Public: login, signup
│   ├── layout.tsx
│   ├── login/
│   │   └── page.tsx        # /login
│   └── signup/
│       └── page.tsx        # /signup
├── (dashboard)/            # Authenticated
│   ├── layout.tsx
│   ├── dashboard/
│   │   └── page.tsx        # /dashboard
│   ├── assets/
│   │   └── page.tsx        # /assets
│   ├── airlock/
│   │   └── page.tsx        # /airlock
│   ├── vault/
│   │   └── page.tsx        # /vault
│   ├── governance/
│   │   └── page.tsx        # /governance
│   └── settings/
│       └── page.tsx        # /settings
```
