## Phase 1.2 — Auth loop (complete)

### Supabase/Resend Auth loop & global session state

- **Log In (`/login`):** Server Action `loginAction` calls `supabase.auth.signInWithPassword`; on success redirects to `/dashboard`, on failure returns generic "Invalid email or password". UI: Shadcn `Button`, `Input`, `useActionState` for errors.
- **Sign Up (`/signup`):** Server Action `signUpAction` calls `supabase.auth.signUp`; redirects to `/login` after signup. Data contract: User table row must be created via DB trigger or post-signup hook.
- **Forgot Password (`/forgot-password`):** Server Action `forgotPasswordAction` calls `supabase.auth.resetPasswordForEmail`; minimalist flow with success message.
- **Log Out:** Sidebar logout button submits form with Server Action `logoutAction`; calls `supabase.auth.signOut()` and redirects to `/login`.
- **Session state:** `AuthProvider` (root layout) consumes Supabase browser client, fetches session on mount, subscribes to `onAuthStateChange`, exposes `session` and `isLoading` via React Context (`useAuth()`). Middleware enforces default-deny; layouts are conditionally effective via middleware redirects (unauthenticated → `(auth)`; authenticated on `/` or `/login` → `/dashboard`).
